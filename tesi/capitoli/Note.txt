\section{Information Architecture e Trovabilita'}
\label{sec:ia}
% =============================================================================
%
% Obiettivo: ancorare il tuo lavoro a una teoria riconosciuta
% (Morville e Rosenfeld) e dimostrare che l'LLM Query Builder
% non e' solo un trucco tecnico, ma una scelta di design motivata.
%

\subsection{Il paradigma classico: navigazione e filtri}
\label{subsec:paradigma_classico}
%
% Paragrafo 1 - L'Information Architecture tradizionale:
%   Introduce il concetto di Information Architecture (IA) secondo
%   Morville e Rosenfeld \cite{Ros12} (gia' nel tuo refs.bib).
%   Riassumi in 4-5 righe i quattro sistemi fondamentali dell'IA:
%   organizzazione, etichettatura, navigazione, ricerca.
%   L'obiettivo e' mostrare al lettore che hai un quadro teorico di
%   riferimento solido, non che stai improvvisando.
%
% Paragrafo 2 - Come funzionava la ricerca nel Datavault PRIMA:
%   Descrivi come si interrogava il Datavault prima del tuo sistema.
%   Probabilmente tramite form con filtri fissi o SPARQL diretto.
%   Quali sono i limiti di questo approccio?
%   - Richiede che l'utente conosca in anticipo i valori possibili
%     (es. il nome esatto del progetto, la sigla del dispositivo).
%   - Non gestisce la sinonimia e la vaghezza del linguaggio naturale.
%   - Non supporta query multi-criterio espresse in modo fluido.
%   Questo crea quella che Morville chiama "trovabilita'" (findability)
%   ridotta: le informazioni esistono ma non sono raggiungibili.
%

\subsection{Il nuovo paradigma: ricerca semantica con LLM}
\label{subsec:paradigma_semantico}
%
% Paragrafo 1 - Cosa cambia con un LLM come interfaccia:
%   Spiega come un Large Language Model usato come "traduttore"
%   dal linguaggio naturale a SPARQL modifichi radicalmente il
%   paradigma di esplorazione dei dati.
%   Il ricercatore non deve piu' adattarsi alla macchina (imparare
%   SPARQL): e' la macchina che si adatta al ricercatore (capisce
%   l'italiano naturale e produce la query corretta).
%   Questo sposta la complessita' dall'utente al sistema.
%
% Paragrafo 2 - Trovabilita' aumentata:
%   Collega esplicitamente il tuo sistema al concetto di "findability"
%   di Morville \cite{Ros12}: l'obiettivo e' che una risorsa digitale
%   presente nel Datavault sia EFFETTIVAMENTE trovabile dall'utente,
%   non solo formalmente accessibile tramite SPARQL.
%   Argomenta che la ricerca semantica aumenta la trovabilita' perche':
%   (a) abbassa la soglia di accesso (chiunque puo' fare una domanda),
%   (b) gestisce la vaghezza e l'ambiguita' del linguaggio,
%   (c) permette query complesse che i form tradizionali non supportano.
%
% Paragrafo 3 - Limiti di questo approccio (essere onesti):
%   Un buon accademico presenta anche i limiti. Cita BREVEMENTE (2-3
%   righe) le limitazioni del paradigma LLM-based: allucinazioni
%   possibili, dipendenza dalla qualita' del prompt di sistema,
%   costo computazionale. Questi verranno approfonditi nel Cap. 4
%   (Valutazione). Usa \cref{chap:valutazione} per rimandare.
%
